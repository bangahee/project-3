# Mini NPU Simulator

## 1. 프로젝트 개요

이 프로젝트는 입력된 패턴이 어떤 필터와 더 유사한지를 판단하는 Python 콘솔 기반 프로그램이다.  
패턴과 필터를 2차원 배열 형태로 표현하고, MAC(Multiply-Accumulate) 연산을 통해 유사도를 계산한다.

사용자는 직접 3×3 필터와 패턴을 입력하거나, JSON 파일에 저장된 데이터를 이용하여 여러 패턴을 자동으로 분석할 수 있다.

python3 main.py

## 3. 구현 요약

### 3.1 MAC 연산

MAC(Multiply-Accumulate) 연산은 입력 패턴과 필터의 같은 위치 값을 곱한 후 모두 더하여 점수를 계산하는 방식이다.

```python
total += pattern[i][j] * filter[i][j]
```

점수가 높을수록 해당 필터와 더 유사하다고 판단한다.

### 3.2 판정 방식

두 필터(Cross, X)의 점수를 비교하여 결과를 결정한다.

* Cross 점수 > X 점수 → Cross
* X 점수 > Cross 점수 → X
* 두 점수 차이가 매우 작은 경우 → UNDECIDED

부동소수점 오차를 고려하여 epsilon 기반 비교를 사용하였다.

```python
abs(score_cross - score_x) < 1e-9
```

### 3.3 라벨 정규화

JSON 데이터에는 다양한 형태의 라벨이 존재할 수 있으므로, 내부적으로 표준 라벨로 변환하여 사용한다.

| 입력 값    | 변환 결과 |
| ------- | ----- |
| "+"     | Cross |
| "cross" | Cross |
| "x"     | X     |

이를 통해 비교 과정에서 오류를 방지하였다.

### 3.4 입력 검증

사용자 입력 및 JSON 데이터에 대해 다음 검증을 수행한다.

* 행/열 개수 확인
* 숫자 형식 검증
* 필터와 패턴 크기 일치 여부 확인

잘못된 입력은 프로그램을 종료하지 않고, 해당 케이스만 FAIL 처리하도록 구현하였다.

### 3.5 성능 분석

MAC 연산의 실행 시간을 측정하기 위해 동일 연산을 10회 반복 실행한 후 평균 시간을 계산하였다.

시간 측정은 입력/출력(I/O) 시간을 제외하고 MAC 연산 함수 실행 구간만 측정하였다.

출력 항목:

* 크기 (N×N)
* 평균 실행 시간 (ms)
* 연산 횟수 (N²)

## 4. 결과 리포트

### 4.1 실행 결과 요약

* 총 테스트 수: 4
* 통과: 4
* 실패: 0

모든 테스트 케이스에서 expected 값과 실제 판정 결과가 일치하였다.

### 4.2 실패가 0인 이유

다음 요소들이 정상적으로 구현되었기 때문이다.

* 라벨 정규화 적용
* epsilon 기반 비교 적용
* 크기 및 형식 검증 수행
* 정확한 MAC 연산 구현

또한 실패 원인은 데이터 문제, 스키마 문제, 수치 비교 문제로 나누어 분석할 수 있으며, 본 프로젝트에서는 해당 문제들이 발생하지 않도록 구현하였다.

### 4.3 시간 복잡도 분석

MAC 연산은 N×N 행렬의 모든 원소를 한 번씩 처리하므로 총 연산 횟수는 N²이다.

따라서 시간 복잡도는 다음과 같다.

```
O(N²)
```

### 4.4 성능 분석 해석

실행 결과, 행렬 크기가 증가할수록 실행 시간이 증가하는 경향을 확인할 수 있었다.

예를 들어:

| 크기    | 연산 횟수 |
| ----- | ----- |
| 3×3   | 9     |
| 5×5   | 25    |
| 13×13 | 169   |
| 25×25 | 625   |

이는 이론적인 시간 복잡도 O(N²)와 일치한다.
